# Patrocinadores

Como um projeto gratuito e de código aberto, o SysIdentPy depende do apoio da comunidade para seu desenvolvimento. Se você trabalha para uma organização que usa e se beneficia do SysIdentPy, por favor considere nos apoiar.

O SysIdentPy __não__ segue a estratégia de lançamento __sponsorware__, o que significa que todos os recursos são liberados ao público ao mesmo tempo. O SysIdentPy é um projeto impulsionado pela comunidade, porém patrocínios ajudarão a garantir sua sustentabilidade.

O principal objetivo dos patrocínios é tornar este projeto sustentável. Sua doação vai para apoiar uma variedade de serviços e desenvolvimento, pois dão aos mantenedores deste projeto tempo para trabalhar no desenvolvimento de novos recursos, correção de bugs, melhoria de estabilidade, triagem de issues e suporte geral.

Leia mais para saber [como se tornar um patrocinador]!

  [como se tornar um patrocinador]: #como-se-tornar-um-patrocinador

## Como se tornar um patrocinador

Obrigado pelo seu interesse em patrocinar! Para se tornar um patrocinador elegível com sua conta do GitHub, visite o [perfil de patrocinador do wilsonrljr], e complete um patrocínio de __$1 por mês ou mais__. Você pode usar sua conta pessoal ou de organização do GitHub para patrocinar.

## Perguntas frequentes

??? question "Não quero mais patrocinar. Posso cancelar meu patrocínio?"

    Sim, você pode cancelar seu patrocínio a qualquer momento! Se você não quiser mais patrocinar o SysIdentPy no GitHub Sponsors, pode solicitar um cancelamento que entrará em vigor no final do ciclo de cobrança. Lembre-se: patrocínios não são reembolsáveis!

    Se você tiver algum problema ou mais perguntas, entre em contato pelo
    wilsonrljr@outlook.com.

??? question "Não queremos pagar pelo patrocínio todo mês. Existem outras opções?"

    Sim. Você pode patrocinar anualmente [mudando sua conta do GitHub para um ciclo de cobrança anual][ciclo de cobrança] ou simplesmente escolher uma doação única.

    Se você tiver algum problema ou mais perguntas, entre em contato pelo
    wilsonrljr@outlook.com.

      [ciclo de cobrança]: https://docs.github.com/en/github/setting-up-and-managing-billing-and-payments-on-github/changing-the-duration-of-your-billing-cycle
